<task type="auto">
  <name>Task 1: Implement API key storage</name>
  <files>src/config/secrets.ts, src/commands/config.ts</files>
  <action>
    Create src/config/secrets.ts with platform-specific secure storage:

    **macOS** (detect via `process.platform === "darwin"`):
    - `storeApiKey(key)`: runs `security add-generic-password -a dgent -s dgent-api-key -w <key> -U` (the -U flag updates if exists)
    - `loadApiKey()`: runs `security find-generic-password -a dgent -s dgent-api-key -w` and returns the password
    - `deleteApiKey()`: runs `security delete-generic-password -a dgent -s dgent-api-key`

    **Linux** (fallback for all non-darwin):
    - Store at `~/.local/share/dgent/.key` as a plain file with 600 permissions
    - `storeApiKey(key)`: write key to file, chmod 0o600
    - `loadApiKey()`: read file
    - `deleteApiKey()`: delete file
    - NOTE: The architecture says "encrypted file" but for Phase 5 shipping speed, plain file with restrictive permissions is acceptable. Log a warning recommending users set ANTHROPIC_API_KEY env var instead on Linux.

    Wire into `dgent config set api-key <value>`:
    - Intercept the `api-key` key in the config set command
    - Call storeApiKey() instead of writing to config.json
    - Print "API key stored securely"

    Also support ANTHROPIC_API_KEY env var as an override (checked first before stored key).

    Export `getApiKey()` that checks env var first, then stored key, returns null if neither.
  </action>
  <verify>`dgent config set api-key test-key-123` stores key. Verify macOS keychain has the entry via `security find-generic-password -a dgent -s dgent-api-key -w`. `getApiKey()` returns the stored value.</verify>
  <done>API key stored in macOS keychain (or Linux file), retrievable via getApiKey(), env var override works</done>
</task>

<task type="auto">
  <name>Task 2: Create Anthropic client module</name>
  <files>src/ai/client.ts, src/ai/index.ts</files>
  <action>
    Install `@anthropic-ai/sdk` as a production dependency.

    Create src/ai/client.ts:
    - `createClient()`: instantiates `new Anthropic({ apiKey: getApiKey() })`. Returns null if no API key available.
    - `callSkill(systemPrompt: string, userContent: string, schema: object)`: calls `client.messages.parse()` with:
      - model from config (`config.ai.model`)
      - system prompt from the skill.md content
      - user content (the diff or commit message)
      - `output_config.format` using `jsonSchemaOutputFormat(schema)`
      - max_tokens: 4096
      - Timeout: 30 seconds
    - Returns the parsed structured output or null on error
    - Wraps in try/catch — AI errors are never fatal

    Create src/ai/index.ts re-exporting from client.ts.

    Do NOT call the API in this task — just build the client. It will be used in 05-02.
  </action>
  <verify>`npm run build` succeeds, client module exports createClient and callSkill</verify>
  <done>Anthropic client module ready, structured output via jsonSchemaOutputFormat, graceful error handling</done>
</task>
