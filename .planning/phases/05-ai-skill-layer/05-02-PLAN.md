  <name>Task 2: Implement rewrite-message rule</name>
  <files>src/rules/rewrite-message.ts, src/ai/index.ts</files>
  <action>
    Create src/rules/rewrite-message.ts:
    - name: "rewrite-message", phase: "commit-msg", type: "fix", defaultEnabled: false
    - This rule is ASYNC — the Rule interface may need updating to support `apply()` returning a Promise
    - apply(input):
      1. Check if AI is enabled in config AND api key is available. If not, skip silently.
      2. Get recent commit messages from git log (last 10): `git log --format=%s -n 10`
      3. Load skill via skill-loader
      4. Call the AI client with:
         - System prompt: skill.md content
         - User content: the commit message + the 10 recent messages as context
         - Schema: `{ rewritten_message: string, changed: boolean, reason: string }`
      5. If Claude returns changed=true, use the rewritten message
      6. If Claude returns changed=false or errors, return input unchanged
    - On any error, return input unchanged with no flags (never block)

    Update Rule interface in src/rules/index.ts:
    - Change `apply` to return `RuleResult | Promise<RuleResult>`
    - Update commit-msg handler to await rule results

    Register in rules array.

    NOTE: No fixtures for this rule — it requires an API call. Testing is manual.
  </action>
  <verify>`npm run build` succeeds, rewrite-message rule is registered but disabled by default, existing fixtures still pass</verify>
