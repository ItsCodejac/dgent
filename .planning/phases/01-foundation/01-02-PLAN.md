    Create src/commands/ directory with one file per command. Each command file exports a function that registers the command on the program. Each handler prints "[command] not implemented yet" and exits 0.

    Commands to stub (from ARCHITECTURE.md Section 8):
    - `dgent init` — "Install global git hooks"
    - `dgent uninstall` — "Remove global hooks"
    - `dgent review` — "Open TUI to review flags"
    - `dgent run <file>` — accepts file arg or stdin via `-`, "Manual pass on a file or stdin"
    - `dgent config` — "Open interactive config editor" (no subcommand)
    - `dgent config set <key> <value>` — "Set a config value"
    - `dgent config list` — "Print current config"
    - `dgent log` — "Show flagged items"
    - `dgent test` — with `--rule <name>` and `--update` options
    - `dgent update` — "Self-update"
    - `dgent hook <type>` — hidden command (not shown in help), accepts "pre-commit" or "commit-msg" plus args. This is what the hook scripts call.

    The `hook` command is critical — it's the entry point for git hooks. It should accept a subcommand (pre-commit or commit-msg) and forward args. Stub for now.

    Build and verify `dgent --help` shows all commands.
