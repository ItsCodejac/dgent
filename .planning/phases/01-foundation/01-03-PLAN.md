    ```
    output: { verbose: false, "log-dir": "~/.local/share/dgent/logs" }
    ```

    Export a TypeScript type `DgentConfig` matching this shape.

    Implement src/config/index.ts with:
    - `getConfigPath()`: returns `~/.config/dgent/config.json` (resolve ~ to os.homedir())
    - `loadConfig()`: reads JSON file, deep-merges with defaults (so new config keys get defaults), returns typed config. If file doesn't exist, returns defaults.
    - `saveConfig(config)`: writes JSON to config path, creating directories if needed (mkdir -p equivalent via fs.mkdir recursive)
    - `getConfigValue(key)`: dot-notation access like "rules.strip-trailers"
    - `setConfigValue(key, value)`: dot-notation set, auto-converts "true"/"false" strings to booleans

    Do NOT use any config library — plain fs read/write with JSON.parse/stringify. Keep it simple.
    Handle the edge case where config file exists but is invalid JSON — log a warning and return defaults.
  </action>

  <done>Config module reads/writes JSON, merges with defaults, handles missing file and invalid JSON gracefully</done>
