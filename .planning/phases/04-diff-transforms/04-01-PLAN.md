<task type="auto">
  <name>Task 1: Implement pre-commit hook handler with partial staging detection</name>
  <files>src/hooks/pre-commit.ts, src/commands/hook.ts</files>
  <action>
    Create src/hooks/pre-commit.ts with:

    `getStagedFiles()`: runs `git diff --cached --name-only --diff-filter=ACM` to get list of staged files (Added, Copied, Modified — not Deleted).

    `hasUnstagedChanges(file)`: runs `git diff --name-only -- <file>` to check if the file has unstaged changes. If yes, this file has mixed staged/unstaged state.

    `handlePreCommit()`:
    1. Get list of staged files
    2. Filter to only text files (skip binaries — check by extension or `file` command)
    3. For each staged file, check hasUnstagedChanges
    4. Split into: safeFiles (fully staged) and mixedFiles (partial staging)
    5. For mixedFiles: skip diff transforms, print warning to stderr listing skipped files
    6. For safeFiles: read file content, run all pre-commit rules, if any rule changed content:
       a. Write modified content back to file
       b. Run `git add <file>` to re-stage
    7. Dry-run mode (DGENT_DRY_RUN=1): show what would change without modifying
    8. Wrap everything in try/catch — never block commit on error

    Wire into src/commands/hook.ts: `dgent hook pre-commit` calls handlePreCommit().

    NOTE: No pre-commit rules exist yet — the handler will iterate an empty list and do nothing. Rules are added in plans 04-02 and 04-03.
  </action>
  <verify>`npm run build` succeeds. `dgent hook pre-commit` runs without error on a repo with staged files. With mixed staging, skipped files are reported.</verify>
  <done>Pre-commit handler detects staged files, identifies partial staging, skips mixed files with warning, runs rule pipeline on safe files, re-stages modified files</done>
</task>

<task type="auto">
  <name>Task 2: Implement first-run consent flow</name>
  <files>src/hooks/consent.ts, src/hooks/pre-commit.ts, src/config/index.ts</files>
  <action>
    Create src/hooks/consent.ts:
    - `hasConsented()`: checks if `~/.config/dgent/consent` file exists
    - `recordConsent()`: creates the consent file
    - `promptConsent()`: prints explanation of what dgent will do to staged files, then reads a line from stdin. If user types "yes" or "y", calls recordConsent(). If anything else, prints "Skipping diff transforms. Run `dgent init --consent` to enable later." and returns false.

    Integrate into handlePreCommit():
    - Before running any diff rules on files, check hasConsented()
    - If not consented: call promptConsent()
    - If consent denied: skip file transforms (still run commit-msg rules)
    - Only prompt once — after consent, it's permanent

    The consent message should explain:
    - dgent will modify staged files and re-stage them
    - Which rules are enabled (list from config)
    - Use --dry-run to preview changes first
    - This is a one-time prompt

    NOTE: For non-interactive environments (CI, scripts), consent should auto-skip without prompting. Detect by checking if stdin is a TTY.
  </action>
  <verify>First run of pre-commit hook shows consent prompt. After consenting, subsequent runs skip the prompt. ~/.config/dgent/consent file exists after consenting.</verify>
  <done>First-run consent flow works, persists to file, auto-skips in non-interactive environments, only prompts once</done>
</task>
