  <name>Task 1: Implement strip-section-headers rule with fixtures</name>
  <files>src/rules/strip-section-headers.ts, test/fixtures/strip-section-headers/*.ts</files>
  <action>
    Create src/rules/strip-section-headers.ts:
    - name: "strip-section-headers", phase: "pre-commit", type: "fix", defaultEnabled: true
    - Detects and removes divider/section header comments:
      - `# --- Section Name ---`
      - `// ===== Section =====`
      - `# ========`
      - `/* --- Section --- */`
      - Any comment line that is purely decorative (only contains dashes, equals, asterisks, and optional text)
    - Only removes in files under a configurable line threshold (default: 100 lines)
    - Pattern: comment marker + optional whitespace + repeating separator chars (-, =, *, #) + optional text + repeating separator chars
    - Also removes blank lines immediately following a removed header (to avoid double-spacing)

    Register in rules array.

    Create fixtures (.ts extension for code files):
    - basic-headers.input.ts / .expected.ts — file with `// --- Init ---` style headers removed
    - dash-and-equals.input.ts / .expected.ts — mixed separator styles
    - large-file.input.ts / .expected.ts — file over 100 lines, headers preserved (expected = input)
    - false-positive-hr.input.ts / .expected.ts — markdown-style hr in a comment (should be removed — it's still a section header)
    - no-headers.input.ts / .expected.ts — clean file (expected = input)
  </action>
  <verify>`dgent test --rule strip-section-headers` passes all fixtures</verify>
  <done>strip-section-headers removes divider comments in small files, preserves large files, all fixtures pass</done>
</task>
