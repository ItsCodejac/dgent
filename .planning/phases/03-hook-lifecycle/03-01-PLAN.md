  <name>Task 1: Implement dgent init with conflict detection</name>
  <files>src/commands/init.ts, src/hooks/install.ts</files>
  <action>
    Create src/hooks/install.ts with:
    - `getHooksDir()`: returns path for dgent's hook directory (~/.config/dgent/hooks/)
    - `isOwnedByDgent(hooksPath)`: checks if the directory contains a `.dgent` marker file
    - `installHooks()`: implements the algorithm from ARCHITECTURE.md Section 3:
      1. Run `git config --global core.hooksPath` to check current setting
      2. If set and not owned by dgent → abort with message showing existing path and manual chaining instructions
      3. If set and owned by dgent → idempotent update (overwrite hook scripts)
      4. If not set → create hooks dir, write hook scripts, set core.hooksPath
    - Write commit-msg hook script:
      ```sh
      #!/bin/sh
      dgent hook commit-msg "$@" || true
      ```
    - Write .dgent marker file in hooks dir
    - Set executable permissions on hook scripts (chmod +x)
    - Print confirmation message

    Wire src/commands/init.ts to call installHooks().

    NOTE: Only install commit-msg hook in this phase. pre-commit hook is added in Phase 4.
  </action>
  <verify>`dgent init` creates hooks directory, writes commit-msg script, sets core.hooksPath. Running `git config --global core.hooksPath` shows the dgent hooks directory. Running `dgent init` again is idempotent.</verify>
  <done>dgent init installs commit-msg hook with conflict detection, is idempotent, prints manual chaining instructions on conflict</done>
</task>
