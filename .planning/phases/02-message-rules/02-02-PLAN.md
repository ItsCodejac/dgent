<task type="auto">
  <name>Task 2: Implement normalize-format rule with fixtures</name>
  <files>src/rules/normalize-format.ts, test/fixtures/normalize-format/*.txt</files>
  <action>
    Create src/rules/normalize-format.ts:
    - name: "normalize-format", phase: "commit-msg", type: "fix", defaultEnabled: true
    - apply(input): enforces conventional commit format IF the message looks like it's attempting conventional commits
    - Detection: if subject line starts with a known type (feat, fix, chore, docs, refactor, test, perf, ci, build, style) followed by optional scope and colon
    - Fixes:
      - Lowercase the type if capitalized: "Feat:" → "feat:"
      - Remove period at end of subject line
      - Ensure single space after colon: "feat:add" → "feat: add"
      - Lowercase first word after colon if uppercase: "feat: Add feature" → "feat: add feature"
    - If message doesn't look like conventional commits, leave it completely alone (return unchanged)
    - This is conservative — only fixes messages that are already attempting the format

    Register in src/rules/index.ts.

    Create fixtures:
    - uppercase-type.input.txt / .expected.txt — "Feat: add feature" → "feat: add feature"
    - trailing-period.input.txt / .expected.txt — "fix: resolve bug." → "fix: resolve bug"
    - missing-space.input.txt / .expected.txt — "feat:add feature" → "feat: add feature"
    - uppercase-description.input.txt / .expected.txt — "fix: Fix the bug" → "fix: fix the bug"
    - already-correct.input.txt / .expected.txt — "feat: add feature" → unchanged
    - not-conventional.input.txt / .expected.txt — "Added a new feature" → unchanged (not conventional format)
    - with-scope.input.txt / .expected.txt — "Fix(auth): Resolve login" → "fix(auth): resolve login"

    Run `dgent test --rule normalize-format`.
  </action>
  <verify>`dgent test --rule normalize-format` passes all fixtures</verify>
  <done>normalize-format fixes conventional commit formatting, leaves non-conventional messages alone, all fixtures pass</done>
